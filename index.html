InView-JS

Check when HTML elements are inside the viewport with IntersectionObserver (to keep test off main thread). You can either add classes for various states or provide your own JS callback function.

Usage

To use, simply insert a script with a src pointing to inview.js, then initialise it with “new ViewTracker('Selector')” where “Selector” is a valid CSS selector.

<script type="text/javascript" src="js/inview.0.1.ug.js"></script> <script type="text/javascript"> (function(){ new ViewTracker('section'); })(); </script>

Optional Parameters

UserOptions [Object]

Specify your own classes by passing an object as the second parameter. If specified, must be the 2nd parameter. Example:

new ViewTracker('Selector', { Hidden : 'Invisible', Partial : 'PeekingOut', Visible : 'FullyInView', });

CallBack [Function]

Specify your own callback function to run custom code when the element comes into view. Can be used as 2nd parameter if “UserOptions” is ommitted. The callback contains 2 parameters. Element - the HTML element. Visibility - A float indicating the level of visibility between 0.0 (fully outside viewport) and 1.0 (fully inside viewport). Example:

new ViewTracker('Selector', function(Element, Visibility){ Element.innerHTML = Visibility; });
